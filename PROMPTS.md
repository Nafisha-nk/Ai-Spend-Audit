# Prompts

## Personalized Summary Prompt

The app uses the prompt generated by `buildSummaryPrompt` in `src/lib/summary.ts`.

```text
Write a clear, specific, finance-literate audit summary in about 100 words.

Rules:
- Do not invent tools, prices, users, sources, or guarantees.
- Be honest if savings are low.
- Mention Credex only if monthly savings exceed $500 or discounted credits are relevant.
- Use plain language for a startup founder or engineering manager.

Audit context:
Team size: {teamSize}
Primary use case: {useCase}
Total current spend: ${totalMonthlySpend}/mo
Total recommended spend: ${totalRecommendedMonthlySpend}/mo
Total savings: ${totalMonthlySavings}/mo, ${totalAnnualSavings}/yr
Credex eligible: {yes/no}

Recommendations:
{per-tool current spend, recommended spend, action, reason}
```

## Why This Prompt

The summary should sound like a careful analyst, not a generic chatbot. The prompt bans invented numbers because the audit math is already done by deterministic code. It also tells the model when Credex should appear, so low-savings users get an honest “you are spending well” message instead of a fake sales pitch.

## What Did Not Work

I did not use an LLM to decide savings because that made the recommendations harder to defend. The model is only used after the math is complete.
